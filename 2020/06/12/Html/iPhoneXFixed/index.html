iPhoneX系列页面底部优化（解决误触小黑边）

  • HTML

随着iPhone刘海屏的出现，给与用户更大的阅读视野的同时，也为网站开发者带来了新的困扰。屏幕顶部刘海屏以及底部手势的“小黑边”都会成为阻挡页面内容的一部分，顶部问题已经由浏览器为我们解决了，那么如何去适配底部呢，本文章将说明具体方法。

一、安全区域

想要解决这个问题，就先得弄明白什么是安全区域。

在异形屏出来之前，屏幕几乎都是方方正正的，不会存在网页被阻挡的情况（你想象一下苹果没适配的话，网页被那丑陋的刘海挡住的样子），也没有配合全面屏手势而出现的”底部小黑边”。在iPhoneX出来之后，为了解决上述问题，苹果在iOS11中带来了全新的特性：viewport-fit和env，以及safe-area安全区域这么一个概念。

安全区域指的是一个可视窗口范围，处于安全区域的内容不受圆角（corners）、齐刘海（sensor housing）、小黑条（Home Indicator）影响，如下图蓝色区域：

二、viewport-fit和env

viewport-fit

ios11新特性，是苹果为了适配iPhoneX及后续刘海屏机型，对viewport meta 标签的一个扩展，用于设置网页在可视窗口的布局方式，可设置三个值：

  • contain: 可视窗口完全包含网页内容
  • cover：网页内容完全覆盖可视窗口
  • auto：默认值，跟 contain 表现一致

env

ios11新特性，Webkit 的一个 CSS 函数，用于向 CSS 插入用户代理定义的变量设定安全区域与边界的距离，有四个预定义的变量：

  • safe-area-inset-left：安全区域距离左边边界距离
  • safe-area-inset-right：安全区域距离右边边界距离
  • safe-area-inset-top：安全区域距离顶部边界距离
  • safe-area-inset-bottom：安全区域距离底部边界距离

env()必须配合 viewport-fit=cover 使用

三、解决方案

可以使用safe-area-inset-bottom来解决小黑边问题，有如下两种思路：

1、在页面添加padding

2、封装成组件，组件高度为env(safe-area-inset-bottom)

本人推荐使用组件的形式解决（考虑灵活度），当然也可以在全局layout中添加。或许有同学会问，这样设置的话，会不会影响到安卓和iPhone678之类的机型？env()仅会在iOS11以上的系统中才会生效，在安卓中不生效就相当于高度为0。iPhone678中env()计算出来的值为0，也就是说，在安卓和iPhone678之类的机型中，env()得到的值可以看做是0，并不会影响到原样式。
